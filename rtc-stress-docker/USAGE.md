# RTC 压测控制台使用指南

访问地址：`http://172.31.11.224:8088/`

---

## 页面入口

打开主页后，顶部有三个入口：

| 入口 | 地址 | 用途 |
|------|------|------|
| Meeting 压测控制台 | `http://172.31.11.224:8088/` | 批量推流压测（多用户进频道） |
| 发流控制台 | `http://172.31.11.224:8088/stream` | 单路推流（调试单个 sample_send_h264_pcm 进程） |
| 频道主发布者与订阅用户控制台 | `http://172.31.11.224:8088/channel-users` | 按频道批量启动主播+订阅者 |

---

## 一、Meeting 压测控制台（`/`）

适用场景：批量模拟多个用户进入同一频道推/拉流。

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `total` | 30 | 总用户数 |
| `uid_start` | 10001 | 起始 UID，依次递增 |
| `user_per_proc` | 6 | 每个进程承载的用户数，总进程数 = total ÷ user_per_proc |
| `wait` | 2 | 每启动一个进程后等待的秒数，控制拉起速度 |
| `cname` | agora0001 | 频道名 |
| `token` | f29d4dc... | Token 或 AppId |
| `local_ap` | 172.31.11.174 | 本地 AP 地址 |
| `local_ap_cert` | ap.1452738.agora.local | AP 的 SNI / Cert |
| `video_high` | high.h264 | 大流视频文件名 |
| `video_low` | low.h264 | 小流视频文件名 |
| `audio` | zxkgf.wav | 音频文件名 |
| `subscribe_high_count` | 0 | 每个用户订阅大流的路数（0=不订阅） |
| `subscribe_low_count` | 0 | 每个用户订阅小流的路数（0=不订阅） |
| `speaker_start` | 0 | 订阅池起始 UID |
| `speaker_end` | 0 | 订阅池结束 UID |
| `pub_audio` | false | 是否同时发布音频 |

### 操作步骤

1. 修改参数（至少确认 `token`、`local_ap`、`cname`）
2. 点击 **启动压测**
3. 下方执行输出框会实时显示进度，例如：
   ```
   [14:02:01] 开始启动 Meeting 压测
   [14:02:01] 目标用户数：30，每进程用户数：6，预计进程数：5
   [14:02:03] 压测启动进度：当前 1/5 个 sample_send_h264_pcm 进程
   [14:02:05] 压测启动进度：当前 2/5 个 sample_send_h264_pcm 进程
   ...
   [14:02:11] 压测进程数已达到目标：5/5
   ```
4. 点击 **刷新状态** 可查看当前进程数和日志文件列表
5. 压测结束后点击 **停止全部压测** 终止所有进程
6. 点击 **打包下载日志** 下载包含压测日志和 SDK 日志的 tar.gz 包

## 二、发流控制台（`/stream`）

适用场景：单路推流调试，验证单个推流进程是否正常工作。

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `token` | f29d4dc... | Token 或 AppId |
| `cname` | agora0001 | 频道名 |
| `uid` | 999999 | 推流用户 UID |
| `audio_input` | zxkgf.wav | 音频文件名 |
| `video_input` | ceshi_1080p.h264 | 视频文件名（可选 ceshi_720p.h264、ceshi_360p.h264、ceshi_180p.h264、low.h264） |
| `ap` | 172.31.11.174 | 本地 AP 地址 |
| `ap_cert` | ap.1452738.agora.local | AP 的 SNI / Cert |
| `hold` | 999999999 | 推流持续时长（秒），默认永久 |

> 后端自动追加固定参数：帧率 15fps、`--a-sub-flag=0`、`--v-sub-flag=0`、`--mute-audio=0`、`--mute-video=0`、`--use-aut=0`，页面不展示这些。

### 操作步骤

1. 确认 `token`、`cname`、`uid`、`ap` 参数
2. 点击 **启动发流**
3. 控制台显示实际执行的命令和进程 PID
4. 点击 **停止全部压测/发流** 终止进程
5. 点击 **打包下载日志** 下载 `stream_<uid>.log` 和 SDK 日志

## 三、频道主发布者与订阅用户控制台（`/channel-users`）

适用场景：模拟多个频道，每个频道有 1 个主发布者 + N 个订阅者。

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `channel_count` | 3 | 频道数量 |
| `user_per_process` | 2 | 每进程用户数 |
| `token` | f29d4dc... | Token 或 AppId |
| `local_ap` | 172.31.11.174 | 本地 AP 地址 |
| `local_ap_cert` | ap.1452738.agora.local | AP 的 SNI / Cert |
| `video_fps` | 15 | 视频帧率 |
| `cname_base` | stress_poc | 频道名前缀，实际频道名为 `stress_poc1000`、`stress_poc1001`… |
| `number_start` | 1000 | 频道编号起始值 |
| `uid_base` | 10000 | UID 基数 |
| `video_high_file` | ceshi_1080p.h264 | 大流视频文件 |
| `video_low_file` | test_360p.h264 | 低流视频文件 |
| `ip` | （空） | 指定 IP，留空则不指定 |
| `port` | （空） | 指定端口，留空则不指定 |
| `wait` | 2 | 每个进程启动间隔秒数 |

### 操作步骤

1. 配置 `channel_count`、`token`、`local_ap` 等参数
2. 点击 **启动频道主发布者与订阅用户**
3. 控制台实时显示进程启动进度
4. 点击 **停止全部压测/发流** 终止所有进程
5. 点击 **打包下载频道任务日志** 下载本次任务日志

## 常用按钮说明

| 按钮 | 说明 |
|------|------|
| 启动压测 / 启动发流 | 后台异步执行，不会阻塞页面 |
| 停止全部压测/发流 | 调用 `stopAll.sh`，终止所有 `sample_send_h264_pcm` 进程 |
| 刷新状态 | 查看当前运行的进程数、任务状态和日志文件列表 |
| 查看最近日志 | 实时滚动显示控制台输出（1 秒刷新一次） |
| 打包下载日志 | 打包压测日志 + SDK 日志为 tar.gz，包含 `manifest.json` 说明文件 |

## 典型测试流程

### 场景 1：验证单路推流是否正常

1. 打开 `/stream`
2. 填入 `token`、`cname`、`uid`、`ap`
3. 点击 **启动发流**，观察控制台是否有正常输出（无报错即表示推流成功）
4. 用 Agora 监控平台或另一个客户端加入同频道，验证是否能看到画面/听到声音
5. 测试完毕点击 **停止全部压测/发流**

### 场景 2：批量压测（30 用户）

1. 打开 `/`（Meeting 压测控制台）
2. 设置 `total=30`、`user_per_proc=6`、`cname=agora0001`，确认 `token` 和 `local_ap`
3. 点击 **启动压测**，等待控制台显示 `压测进程数已达到目标：5/5`
4. 通过 **刷新状态** 监控运行状态
5. 压测结束后点击 **停止全部压测**，再点击 **打包下载日志** 获取日志分析

### 场景 3：多频道压测（3 个频道，每频道 1 主播 + 订阅者）

1. 打开 `/channel-users`
2. 设置 `channel_count=3`、`cname_base=stress_poc`、`number_start=1000`
3. 点击 **启动频道主发布者与订阅用户**
4. 观察进度，完成后下载频道任务日志

---

## 注意事项

- 所有操作在容器内执行，**停止容器会终止所有压测进程**
- 同一时间只能运行一个 Meeting 压测任务，启动新任务前请先点击 **停止全部压测**
- 发流页面可以启动多个独立的发流进程（不同 UID）
- 日志文件保存在容器内 `/home/bbt/` 目录下，建议及时点击 **打包下载日志** 保存
