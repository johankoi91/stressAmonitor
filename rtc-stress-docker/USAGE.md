# RTC 压测控制台使用指南

访问地址：`http://172.31.11.224:8088/`

---

## 打开页面后看到什么

打开主页，顶部蓝紫色横幅里有两个链接：

```
进入发流页面          → /stream
进入频道主发布者与订阅用户页面  → /channel-users
```

页面主体分两部分：
- 上半部分：**压测参数表单**（填写配置）
- 下半部分：**执行输出框**（黑色背景，显示运行日志）

---

## 一、Meeting 压测控制台（主页 `/`）

用途：批量模拟多个用户同时进入一个频道推流，测试频道承载能力。

### 怎么填

页面上有一个 4 列的参数表单，逐项说明如下：

**基础配置（必填）**

| 字段名 | 填什么 | 示例 |
|--------|--------|------|
| `total`（总用户数） | 本次压测要起多少个虚拟用户 | `30` |
| `cname`（频道名） | 要压测的频道名称，和业务侧保持一致 | `agora0001` |
| `token`（Token/AppId） | 填入项目的 AppId 或鉴权 Token | `f29d4dc623c741a3ad38ccaf27a8d2e2` |
| `local_ap`（Local AP） | 部署在内网的 AP 服务器 IP | `172.31.11.174` |
| `local_ap_cert`（AP Cert/SNI） | AP 的域名/证书 SNI | `ap.1452738.agora.local` |

**进程控制（可保持默认）**

| 字段名 | 默认值 | 说明 |
|--------|--------|------|
| `uid_start`（起始 UID） | `10001` | 第一个虚拟用户的 UID，后续依次 +1 |
| `user_per_proc`（每进程用户数） | `6` | 每个底层进程承载几个用户，`total=30`、`user_per_proc=6` → 共起 5 个进程 |
| `wait`（进程间隔秒） | `2` | 每启动一个进程后等待几秒再启动下一个，避免瞬间并发过高 |

**推流内容（可保持默认）**

| 字段名 | 默认值 | 说明 |
|--------|--------|------|
| `video_high`（大流文件） | `high.h264` | 大流视频文件名，容器内有 `ceshi_1080p.h264`、`ceshi_720p.h264` 可选 |
| `video_low`（小流文件） | `low.h264` | 小流视频文件名 |
| `audio`（音频文件） | `zxkgf.wav` | 音频文件名，保持默认即可 |
| `pub_audio`（是否发音频） | `false` | 下拉选 `true` 表示同时推音频流 |

**订阅配置（不需要订阅则保持 0）**

| 字段名 | 默认值 | 说明 |
|--------|--------|------|
| `subscribe_high_count` | `0` | 每个虚拟用户订阅几路大流 |
| `subscribe_low_count` | `0` | 每个虚拟用户订阅几路小流 |
| `speaker_start` | `0` | 被订阅的用户 UID 起始值 |
| `speaker_end` | `0` | 被订阅的用户 UID 结束值 |

### 填写示例（30 用户压测）

```
total            = 30
uid_start        = 10001
user_per_proc    = 6
wait             = 2
cname            = agora0001
token            = 你的AppId或Token
local_ap         = 172.31.11.174
local_ap_cert    = ap.1452738.agora.local
video_high       = ceshi_1080p.h264
video_low        = low.h264
audio            = zxkgf.wav
pub_audio        = false
（其余订阅字段保持 0）
```

### 操作步骤

1. 按照上面填好参数
2. 点击蓝色 **「启动压测」** 按钮
3. 下方黑色输出框会开始滚动显示日志，类似：
   ```
   [14:02:01] 开始启动 Meeting 压测
   [14:02:01] 目标用户数：30，每进程用户数：6，预计进程数：5
   [14:02:03] 压测启动进度：当前 1/5 个进程
   [14:02:05] 压测启动进度：当前 2/5 个进程
   [14:02:09] 压测启动进度：当前 4/5 个进程
   [14:02:11] 压测进程数已达到目标：5/5
   ```
4. 看到「达到目标」说明所有进程已成功启动
5. 点击 **「刷新状态」** 可以随时查看当前运行进程数
6. 压测结束后点击红色 **「停止全部压测」** 终止所有进程
7. 点击 **「打包下载日志」** 下载日志压缩包（含业务日志 + SDK 日志 + 索引文件）

## 二、发流控制台（`/stream`）

用途：单独启动一路推流进程，用于调试验证推流链路是否通畅。适合先用单路确认环境正常，再做批量压测。

### 怎么填

页面参数比主页少，填写更简单：

| 字段名 | 填什么 | 示例 |
|--------|--------|------|
| `token`（Token/AppId） | 同主页，填 AppId 或 Token | `f29d4dc623c741a3ad38ccaf27a8d2e2` |
| `cname`（频道名） | 要进入的频道名 | `agora0001` |
| `uid`（UID） | 这路推流的用户 ID，随意设一个不冲突的数字 | `999999` |
| `audio_input`（音频文件） | 容器内的音频文件名 | `zxkgf.wav` |
| `video_input`（视频文件） | 容器内的视频文件名，可选分辨率如下 | `ceshi_1080p.h264` |
| `ap`（Local AP） | 内网 AP 服务器 IP | `172.31.11.174` |
| `ap_cert`（AP Cert/SNI） | AP 证书 SNI | `ap.1452738.agora.local` |
| `hold`（保持时长） | 推流持续多少秒，填 `999999999` 表示一直推 | `999999999` |

**可用视频文件：**

| 文件名 | 分辨率 |
|--------|--------|
| `ceshi_1080p.h264` | 1080p |
| `ceshi_720p.h264` | 720p |
| `ceshi_360p.h264` | 360p |
| `ceshi_180p.h264` | 180p |
| `low.h264` | 低码率小流 |

> 帧率、订阅标志等参数由后端固定设置，页面不需要填写。

### 填写示例（调试单路推流）

```
token       = 你的AppId或Token
cname       = agora0001
uid         = 999999
audio_input = zxkgf.wav
video_input = ceshi_1080p.h264
ap          = 172.31.11.174
ap_cert     = ap.1452738.agora.local
hold        = 999999999
```

### 操作步骤

1. 填好上面参数
2. 点击绿色 **「启动发流」** 按钮
3. 输出框显示类似以下内容说明推流已启动：
   ```
   {
     "accepted": true,
     "command": "LD_LIBRARY_PATH=./ nohup ./sample_send_h264_pcm ...",
     "meta": { "uid": 999999, "cname": "agora0001", ... }
   }
   ```
4. 用 Agora 控制台或另一个终端客户端加入同频道，验证是否收到画面/声音
5. 确认正常后点击 **「停止全部压测/发流」** 结束推流
6. 点击 **「打包下载日志」** 下载推流日志（`stream_999999.log`）

## 三、频道主发布者与订阅用户控制台（`/channel-users`）

用途：模拟多个频道并发，每个频道有一个主发布者推流 + 若干订阅用户拉流，适合测试多频道并发场景。

### 怎么填

| 字段名 | 填什么 | 示例 |
|--------|--------|------|
| `channel_count`（频道数量） | 同时跑几个频道 | `3` |
| `user_per_process`（每进程用户数） | 每个进程里有几个用户 | `2` |
| `token`（Token/AppId） | AppId 或 Token | `f29d4dc623c741a3ad38ccaf27a8d2e2` |
| `local_ap`（Local AP） | 内网 AP 地址 | `172.31.11.174` |
| `local_ap_cert`（AP Cert/SNI） | AP 证书 SNI | `ap.1452738.agora.local` |
| `video_fps`（视频帧率） | 推流帧率 | `15` |
| `cname_base`（频道名前缀） | 频道名的前缀，实际频道名 = 前缀 + 编号 | `stress_poc` |
| `number_start`（频道编号起始） | 编号从几开始，3 个频道则为 `stress_poc1000`、`stress_poc1001`、`stress_poc1002` | `1000` |
| `uid_base`（UID 基数） | 各频道的 UID 以此为基础计算 | `10000` |
| `video_high_file`（大流视频文件） | 主发布者推流的视频文件 | `ceshi_1080p.h264` |
| `video_low_file`（低流视频文件） | 低流文件名 | `test_360p.h264` |
| `ip`（指定 IP） | 可空，不填则不指定 | （留空） |
| `port`（指定端口） | 可空，不填则不指定 | （留空） |
| `wait`（等待秒数） | 每启动一个进程后的等待秒数 | `2` |

### 填写示例（3 个频道）

```
channel_count   = 3
user_per_process = 2
token           = 你的AppId或Token
local_ap        = 172.31.11.174
local_ap_cert   = ap.1452738.agora.local
video_fps       = 15
cname_base      = stress_poc
number_start    = 1000
uid_base        = 10000
video_high_file = ceshi_1080p.h264
video_low_file  = test_360p.h264
ip              = （留空）
port            = （留空）
wait            = 2
```

实际启动后频道名为：`stress_poc1000`、`stress_poc1001`、`stress_poc1002`

### 操作步骤

1. 填好参数
2. 点击橙色 **「启动频道主发布者与订阅用户」** 按钮
3. 输出框滚动显示启动进度
4. 测试完毕点击 **「停止全部压测/发流」**
5. 点击 **「打包下载频道任务日志」** 下载日志

---

## 按钮说明

| 按钮 | 说明 |
|------|------|
| **启动压测 / 启动发流** | 提交参数，后台异步执行，页面不会卡住 |
| **停止全部压测/发流** | 强制终止容器内所有 `sample_send_h264_pcm` 进程 |
| **刷新状态** | 查看当前进程数、任务状态、日志文件列表 |
| **查看最近日志** | 切换输出框为实时控制台模式，每秒自动刷新 |
| **打包下载日志** | 把业务日志 + SDK 日志打包成 `.tar.gz` 下载到本地 |

## 常见问题

**Q：启动后输出框一直显示「请求中...」没有日志？**
点击 **「查看最近日志」** 切换到实时日志模式。

**Q：启动了但进程数一直是 0？**
检查 `local_ap`、`token`、`cname` 是否填写正确，然后点击 **「打包下载日志」** 查看 SDK 错误日志。

**Q：想重新跑一次压测，之前的进程还在怎么办？**
先点击 **「停止全部压测」** 清除旧进程，再重新填参数启动。

**Q：容器重启后日志还在吗？**
容器重启后 `/home/bbt/` 里的日志文件会丢失，压测结束后务必先点击 **「打包下载日志」** 保存到本地。
